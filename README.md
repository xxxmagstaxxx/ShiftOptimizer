サークル (東大装い喫茶研究会) 用。
Googleフォームの内容からシフト希望を取り出し、そのシフト希望から最適なシフトを作るプログラムです。

前準備

1. Gurobi Optimizerをインストール (東大生なら無料で使える)
2. GeminiのAPIキーを取得 (無料版で良い)
3. Pythonの各種ライブラリをインストール

手順

1. settings.iniに情報を記入 (APIキーなど)
2. shift_query.csvにGoogleフォームの内容を正しいフォーマットになるようにコピー
3. shift_convert.pyを実行 (Googleフォームの内容からシフト希望を取り出すプログラム)
4. shift_shaped.csvが生成され、場合に応じて手直し
5. shift_optimize.pyを実行 (シフト希望から最適なシフトを作るプログラム)
6. shift_best.csvが生成され、最適なシフトが出力

settings.iniの内容について

api: GeminiのAPIキー (デフォルトでは****と書いています)  
model_name: Geminiのモデル (無料版ではgemini-3-flash-previewが良いと思います)  
query: Googleフォームの内容を記入したcsvに対するパス  
shaped: シフト最適化に必要なデータが入ったcsvに対するパス  
best: 最適なシフトが出力されるcsvに対するパス  
time: 数理最適化ソルバーを動かす秒数  

shift_query.csvの内容について

days: 開催日数  
n_people: 参加人数  
3行目: 各日にちに対し、開始時刻と終了時刻を表記 *1  
4行目: どの列が何を表しているのかを表記 *2  
5行目以降: 各行が各人の希望を表している  

*1 時刻は時の2倍の整数値として表現している。例えば11時は22、13時半は27というように表現している。  
*2 nameはdiscordの名前、name2が出店で使う名前を表している。また、day1は1日目の出店に関するシフト希望を表しおり、day2なども同様である。

shift_shaped.csvの内容について

days: 開催日数  
n_people: 参加人数  
alpha: 人によってシフト時間の合計が偏らないようにするための目的関数に関する重み  
beta: 人によって屋外のシフト時間の合計が偏らないようにするための目的関数に関する重み  
gamma: 人によって屋内のシフト時間の合計が偏らないようにするための目的関数に関する重み  
delta: 屋内企画に欲しい人数に近づけるための目的関数に関する重み  
epsilon: 屋外企画に欲しい人数に近づけるための目的関数に関する重み  
zeta: 屋内企画と屋外企画の行き来を少なくするための目的関数に関する重み  
eta: 短すぎるシフトを避けるための目的関数に関する重み  
10行目: 各日にちに対し、開始時刻と終了時刻を表記  
11行目以降の5列ある部分: 各時刻の必要人数と希望人数を表記 *1  
それ以降: 各人のシフト希望を表記 *2  
更にそれ以降: 追加する線形制約を表記 *3  

*1 1列目は時刻、2列目はその時刻での屋内企画の必要人数、3列目は屋外企画の必要人数、4列目は屋内企画の希望人数、5列目は屋外企画の希望人数である。また、1日目のデータを開始時刻から終了時刻まで上から表記した後に2日目のデータを開始時刻から表記する、というような形で早い日にちから表記されている。また、ここでいう時刻は正確には時間区間であり、例えば22は11時から11時半を表している。  
*2 1列目はdiscordの名前、2列目は出店で使う名前である。3列目以降では、まず何日目に出るかを書き、企画全体での参加可能時間と屋外企画の参加可能時間をこの順で表記する。その後、その人が参加可能な時間区間を表記していき、時間区間が2つ以上ある場合は連続してこれらの時間区間を表記する。これをその人が参加するそれぞれの日にちで行う。  
*3 まず、数理最適化問題に新たに追加できる制約は線形制約のみである。まず、変数が入った項を左辺に、定数項を右辺に移項させる。1列目は変数が入った式<=定数項という形の線形制約なら0、変数が入った式>=定数項という形の線形制約なら1、変数が入った式=定数項という形の線形制約なら2とする。2列目は定数項である。3列目以降は、変数が入ったそれぞれの項に対し、係数と変数名 (*4) を順番に書いていく。  
*4 変数名は以下の通りである。  
x[p,d,t]は(p+1)人目の人が{d+1}日目の時間帯tにシフトが入るかどうか(01変数で1のとき入る)  
y[p,d,t]は(p+1)人目の人が{d+1}日目の時間帯tに屋外企画のシフトが入るかどうか(01変数で1のとき入る)  
us[p,d]は(p+1)人目の人の{d+1}日目のシフトの開始時刻  
ue[p,d]は(p+1)人目の人の{d+1}日目のシフトの終了時刻  
vs[p,d]は(p+1)人目の人の{d+1}日目のシフトの屋外企画の開始時刻  
ve[p,d]は(p+1)人目の人の{d+1}日目のシフトの屋外企画の終了時刻

shift_best.csvの内容について

1列目はdiscordの名前、2列目は出店で使う名前である。3列目以降はそのままである。
